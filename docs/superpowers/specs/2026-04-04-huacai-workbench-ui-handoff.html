华彩系统 UI 改版交接简报

给 AI coding agent 的低 token 版本。目标是先建立统一 UI 基线，不扩散到全部业务页。

专业作业台 高密度但分层 A1 钴蓝企业版 Workbench First

严格范围

  • frontend/src/style.css
  • frontend/src/layout/AppLayout.vue
  • frontend/src/views/dashboard/WorkbenchView.vue

不要顺手重构其他业务页，不要新增复杂依赖，不要引入图表库。

布局基线

  • 壳层：左导航 + 顶部页头 + 主内容区
  • 工作台：标题区 + 统计卡 + 主体双栏
  • 主体：左侧主列表，右侧待办/趋势/快捷入口
  • 抽屉承载新增编辑，弹窗承载确认动作

比例规则

  • 页面主标题：26px
  • 区块标题：16px
  • 正文：14px
  • 标签和辅助信息：12px
  • 表格正文：13px
  • 按钮高度：34px 到 36px
  • 按钮和输入框圆角：10px

组件规则

  • 统计卡只保留 3 到 4 个
  • 只有关键卡片能用主色弱底
  • 主按钮钴蓝实底，次按钮白底浅边框
  • 列表采用顶部统计条 + 折叠筛选 + 表格 + 分页
  • 表格允许高密度，但不能拥挤

Do / Don't

Do
  • 让左侧主列表成为页面视觉中心
  • 让图表和快捷入口服务主流程
  • 用细边框、轻阴影、小一档字体建立精致感
  • 让激活态、主按钮、关键数字统一用钴蓝
Don't
  • 不要继续用暖米色和棕橙做主基调
  • 不要大按钮、大圆角、厚卡片、强装饰
  • 不要让图表比列表更抢眼
  • 不要把复杂编辑流程跳转到新页面

验收

  • npm run build 通过
  • 页面不再出现暖米色和棕橙主基调
  • 字体、按钮、卡片比例明显更收敛
  • Workbench 已经具备后续扩展为真实业务页的模板价值
