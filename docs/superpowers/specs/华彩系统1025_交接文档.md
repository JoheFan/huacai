## 5 个 Review Findings 修复说明

### 1. 权限注册缺失 ✅
完整注册所有新模块到 ModuleAccessRegistry：
- ASSIGNABLE_MODULE_KEYS: 添加 `loan-orders-self`, `loan-orders-bank`, `increment-details`, `increment-overview`, `employee-performance`
- ALL_PAGE_PERMISSIONS: 添加 `/loan-orders-self`, `/loan-orders-bank`, `/increment-details`, `/analysis/increment-overview`, `/analysis/performance`
- BUSINESS_API_PREFIXES: 添加 `/api/v1/analysis/*` 映射
- API_REQUIRED_PAGE_PERMISSIONS: 添加 analysis 各端点映射
- LEGACY_MODULE_PAGE_PERMISSIONS / PAGE_PERMISSION_MODULE_CODES: 完整注册

### 2. 借贷拆分仍是旧聚合模型 ✅
- 新建 `loan_customer_summary` 表：持久化存储银行口径字段（totalIncrementAmount, incrementCount, yearsTerm, channelRate, channelFee, referrer, selfTotalLoanAmount, bankTotalLoanAmount）
- `pageOrderOverview` 优先从 `loan_customer_summary` 读取，缺失时兼容运行时聚合
- 添加客户汇总 CRUD 接口（get/save/deleteCustomerSummary）
- `LoanOrderOverviewVO` 添加银行口径字段
- 前端 `LoanOrderOverviewRecord` 添加对应字段

### 3. DailyIncrement 信任客户端传父级元数据 ✅
- `createDailyIncrement` / `updateDailyIncrement` 先调用 `getIncrementDetailOrThrow` 加载父记录
- 父记录不存在时抛 `BusinessException`
- `copyDailyIncrementToEntity` 用父记录的 `customerName` 和 `incrementDate` 覆盖 DTO 值

### 4. Analysis 模块 not found 返回 500 ✅
所有"记录不存在"场景改为 `BusinessException`：
- `getIncrementSummaryDetail` / `updateIncrementSummary`
- `getIncrementDetailDetail` / `updateIncrementDetail`
- `getDailyIncrementDetail` / `updateDailyIncrement`
- `getEmployeePerformanceDetail` / `updateEmployeePerformance`

### 5. V2_100 migration 非幂等 ✅
使用 `INFORMATION_SCHEMA.COLUMNS` 检查列是否存在，仅列不存在时执行 ALTER TABLE。兼容重复执行。

## 业务说明

1. **权限**: 新模块完整接入后端权限注册，前后端 moduleKey / path / pagePermission 一致
2. **借贷管理**: 客户汇总台账支持手工维护，银行口径字段持久化存储
3. **跟进记录**: `follower_name` 手动输入，非用户关联
4. **增量总览**: 新增时自动计算 12 个月合计
5. **员工绩效**: 选择员工后自动回填部门名称
