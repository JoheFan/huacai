# 华彩系统 UI 改版交接简报

## 目标

把当前前端后台从“原型骨架风格”升级为可继续开发的 `专业作业台` 风格，第一阶段只建立统一 UI 基线，不扩散到全部业务页。

## 严格范围

本轮只允许修改：

- `frontend/src/style.css`
- `frontend/src/layout/AppLayout.vue`
- `frontend/src/views/dashboard/WorkbenchView.vue`

不要顺手重构其他业务页面，不要新增复杂依赖，不要引入图表库。

## 视觉基线

- 风格：`专业作业台`
- 密度：`高密度但分层`
- 配色：`A1 钴蓝企业版`
- 主色：`#1E40AF`
- 主色 hover：`#1D4ED8`
- 主色高亮：`#3B82F6`
- 主色弱底：`rgba(59, 130, 246, 0.12)`
- 背景：`#F8FAFC`
- 面板：`#FFFFFF`
- 次级面板：`#F6F9FF`
- 主文字：`#132238`
- 次文字：`#617186`
- 边框：`rgba(148, 163, 184, 0.24)`
- 警告：`#D97706`
- 危险：`#D9485F`
- 成功：`#0F766E`

## 比例规则

- 页面主标题：`29px`
- 区块标题：`18px`
- 正文：`15px`
- 标签和辅助信息：`13px`
- 表格正文：`14px`
- 按钮文字：`13px`
- 按钮高度：`38px` 到 `40px`
- 大面板圆角：`20px`
- 普通卡片圆角：`16px`
- 筛选区圆角：`12px`
- 按钮和输入框圆角：`10px`

## 布局规则

### AppLayout

- 左侧导航更精致，不要过宽
- 激活菜单使用钴蓝弱底 + 蓝色文字
- 顶部保留当前页标题和少量动作
- 用户区和页面动作区不要抢主标题

### Workbench

- 结构固定为三层：
  - 顶部：标题、说明、主操作
  - 中部：3 到 4 个统计卡
  - 主体：左侧主列表区 + 右侧辅助区
- 左侧主区必须是主视觉中心
- 右侧辅助区承载回款趋势、今日待办、快捷入口
- 图表不能抢过列表
- 快捷入口要轻，不要做大块按钮墙

## 列表和弹层规则

- 列表骨架：`顶部统计条 + 折叠筛选 + 表格 + 分页`
- 默认只展示核心筛选项
- 复杂筛选放进“展开筛选”
- 新增/编辑/补录使用抽屉
- 删除/提交/状态确认使用弹窗
- 不跳新页面做复杂编辑，优先在当前上下文完成

## 组件要求

- 统计卡只允许 3 到 4 个
- 只有最关键的卡片可以用主色弱底
- 主按钮用钴蓝实底
- 次按钮用白底浅边框
- 标签必须用“文字 + 颜色”，不能只靠颜色
- 表头弱底处理
- 表格支持高密度，但不能拥挤
- `1024px` 下允许表格横向滚动

## 验收

- `npm run build` 通过
- 页面不再出现暖米色和棕橙主基调
- 字体、按钮、卡片比例明显比现在更收敛
- 工作台已经具备“可继续扩展为真实业务页”的模板价值
- 视觉上是成熟企业后台，不是展示型大屏，也不是偏灰旧后台
