# 华彩系统原型表格字段差异清单

日期：2026-04-15

范围说明：
- 本清单只对照当前已开发页面与 `/原型图` 中的对应原型。
- 本清单不包含工作台。
- 结论分三类：
  - `要补`：当前数据模型已具备或仅需小幅扩 VO/列表展示即可补齐。
  - `确认不做`：明显超出一期收口范围，或属于当前未启用模块/分析口径。
  - `模型改动`：当前实现和原型已经不是同一张表，不能靠简单补列解决。

## 收口建议

建议先做“可直接补齐”的部分，再单独确认“模型改动”的页面，不要混在一轮里做。

### A. 可以直接补的字段

- 客户管理：`统一社会信用代码 / 推荐人 / 推荐人返点 / 业务状态`
- 商机管理：`联系方式 / 企业名称 / 统一社会信用代码 / 备注`
- 借贷管理：`客户ID / 理财定存黄金 / 垫还金额 / 笔数 / 备注`
- 还款明细：`序号 / 备注`，并统一字段文案
- 系统用户：`角色名称 / 创建时间`
- 组织管理：补 `remark / sortNo` 的可见性

### B. 本轮建议确认不做

- 客户管理：`测试日期 / 测试额度 / 流量 / 综合评分 / 龙信商`
- 借贷管理（银行）：`总增量金额 / 增量笔数 / 几年期`
- 还款明细（银行）：`剩余期数`

### C. 必须单独确认的模型改动

- 商机管理：`最近一次跟进时间 / 跟进人 / 跟进记录`
- 借贷管理：整页是“单笔借贷单”还是“客户借贷汇总”
- 还款明细：整页是“全局还款列表”还是“客户下钻详情”
- 角色管理：当前角色实体表是否继续保留
- 组织管理：继续用树还是回退表格
