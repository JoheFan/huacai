默认假设（用于消解 V0.6 中仍未完全定稿的点）：
1. 转正审批节点顺序先按固定链路实现：
   部门审核 -> 人事审核 -> 公司审批 -> 行政审批 -> 办结
2. 调薪审批节点顺序先按固定链路实现：
   部门审核 -> 人事审核 -> 分管领导 -> 校领导 -> 财务办理 -> 办结
3. 调薪“当前薪资”默认读取：
   该员工最近一条有效工资标准；如果不存在，禁止提交并提示先维护工资标准
4. 调薪后薪资：
   当前薪资 + 申请调薪幅度，自动计算，只读
5. 员工档案“是否开通系统账号=是”时：
   保存成功后自动创建或更新系统用户，默认角色为普通用户
6. 导入档案、导入考核、同步、导出：
   如果模板或上游来源在 V0.6 里未明确，不要瞎造格式
   - 页面入口可保留
   - 但如果后端模板和规则未能在本轮合理定义，允许按钮置灰，并在 handoff 文档明确说明
7. “快速处理”：
   当前仅在员工档案列表 `在职` 页签显示；若具体动作仍无法从原型或文档确定，可先保留空菜单/禁用态，不要伪造业务动作
8. `ManagementRecordView.vue`：
   V0.6 没有把它定义成人事一级交付目标，如无必要不要单独扩成主模块；管理记录应优先通过转正/调薪/档案变动留痕实现
