# 华彩组织管理树节点 UI 收口设计

## 1. 目标

- 只优化 `组织管理` 页面中的树节点区域
- 同时解决三个问题：
  - 信息排布混乱
  - 操作区无法稳定对齐
  - 父子层级视觉不清晰
- 不修改页头、顶部操作卡片、抽屉结构
- 不改变当前系统的整体 UI 基线，只做局部收口

## 2. 问题归因

当前树节点使用 `el-tree` 自定义节点模板，但节点内部是“左侧内容 + 右侧操作”的自由流布局，且左侧元信息为可换行的连续文本。

这会带来三个直接问题：

1. 单个节点高度不可控，父节点元信息会向下挤压，视觉上和子节点层级叠在一起。
2. 右侧操作区跟随内容高度漂移，不同节点之间按钮列不齐。
3. 虽然真实数据树是正确的，但用户在视觉上无法快速区分“节点内容”与“子节点内容”。

## 3. 方案选择

本次采用“轻量树行收口”：

- 保留 `el-tree`、现有接口和现有交互
- 重排单个节点内部布局
- 增强层级引导线和缩进节奏
- 用更稳定的内容分组替代当前自由换行元信息
- 不做重卡片、不做明显装饰，只做局部秩序优化

不采用树表重做，也不新增复杂筛选、拖拽、折叠逻辑。

## 4. 目标结构

每个树节点改为更轻量的双层结构：

### 4.1 标题区

- 显示 `组织名称`
- 紧随其后显示：
  - `组织类型`
  - `状态`
- 标题区权重最高，用于让用户先识别“这是哪个组织”

### 4.2 元信息区

- 使用固定节奏的信息项，而不是一整段连续文本
- 展示：
  - 上级机构
  - 是否管理机构
  - 排序
  - 备注
- 在桌面端保持紧凑横向排布，必要时换行
- 在窄屏下自然换为更松的纵向换行

### 4.3 操作区

- 固定在节点右侧
- 包含现有 4 个动作：
  - 新增子组织
  - 编辑
  - 停用/启用
  - 删除
- 按钮组保持轻量文本按钮形态，整体右对齐；在窄屏下换到节点下方

## 5. 层级视觉策略

### 5.1 保留原生树交互

- 保留 `el-tree` 的展开箭头
- 保留当前树形缩进逻辑

### 5.2 增强层级识别

- 保留简单的节点留白，不做显著卡片包裹
- 节点 hover 仅做轻微强调，不影响整棵树
- 使用轻量纵向引导线或缩进辅助线，让父子层级更容易追踪

### 5.3 权重控制

- 根节点与子节点不靠颜色区分，而靠：
  - 名称字号/字重
  - 间距
  - 行内信息节奏
- 不引入新的视觉主题，不改现有蓝灰白基线

## 6. 视觉与交互约束

- 颜色继续沿用当前系统语义色
- 不引入夸张渐变、阴影或风格跳变
- 保留现有按钮文案和状态语义
- 保证按钮点击热区充足
- 保证树节点在多级嵌套时仍然不发生文本压叠

## 7. 实现边界

仅修改：

- `frontend/src/views/system/org/OrgManageView.vue`

不修改：

- 路由
- layout
- 接口
- 后端树结构
- 组织抽屉字段定义

## 8. 验收标准

完成后应满足：

1. 父节点内容与子节点内容不再出现视觉重叠
2. 操作区在多节点之间对齐稳定
3. 组织名称、标签、元信息层级清晰
4. 多级树展开时仍可快速辨认父子关系
5. 页面整体风格仍与当前系统一致
