# design.md

## 项目名称
华彩经营管理平台 UI 设计规范（2026 浅蓝企业 SaaS 风格）

---

## 1. 设计目标

本设计规范用于指导 Agent、设计师和前端工程师，在不改变核心业务信息架构的前提下，将页面统一升级为更现代、更精致的 **2026 企业级浅蓝科技风**。

目标关键词：

- 浅蓝色
- 企业级
- 科技感
- 通透
- 克制
- 精致
- 高级感
- 大留白
- 轻玻璃感
- 数据化
- B 端专业感

整体目标不是做“炫技型视觉”，而是做 **高完成度、可落地、适合真实业务系统长期使用** 的后台界面。

---

## 2. 总体视觉原则

### 2.1 风格定位
这是一个偏 **高端 SaaS 管理后台** 的视觉体系，不是营销官网，也不是消费级产品 UI。

应具备以下气质：

- 稳定、专业、可信赖
- 明亮但不过分花哨
- 有科技感，但不能赛博或过度未来化
- 有高级感，但不能牺牲可读性和业务效率

### 2.2 审美方向
整体视觉参考：

- 浅蓝白企业 SaaS
- 轻拟态 / 微玻璃感
- 柔和阴影
- 更大的圆角体系
- 更轻的边框
- 更少噪音
- 更统一的图标语言
- 更细腻的层次和高光

### 2.3 还原要求
当 Agent 基于设计图或截图做页面重构时：

1. **优先保留原有信息架构**
2. **优先保留原有业务文案**
3. 允许优化布局、间距、图标、层次、装饰
4. 不随意删减模块
5. 同一产品内多个页面必须使用 **同一套设计语言**

## 3. 色彩系统

### 3.1 主色
- Primary Blue: `#4A85FF`
- Primary Blue Deep: `#3167F3`
- Primary Blue Soft: `#8BB6FF`

### 3.2 页面背景
- Background 1: `#EEF4FF`
- Background 2: `#F8FBFF`

推荐背景做法：
- 大面积使用浅蓝白渐变
- 可叠加极轻微光晕或顶部光轨
- 背景装饰不影响正文识别

### 3.3 文字颜色
- 主文字：`#122D5F`
- 次级文字：`#7085AD`
- 弱化文字：`#95A7C4`

### 3.4 边框颜色
- 主边框：`#DBE8FF`
- 次边框：`#CDDFFF`
- 分割线：`#EDF3FF`

### 3.5 功能色
- 成功 / 低优先：`#18A69D`
- 警示 / 中提醒：`#DB7D14`
- 高优先 / 风险：`#EF7386`

### 3.6 配色原则
- 主界面必须以浅蓝、白、灰蓝为核心
- 不使用高饱和荧光色
- 彩色只用于强调按钮、状态、优先级和少量图标
- 大面积内容区必须保持克制、通透、可读

---

## 4. 圆角、边框、阴影

### 4.1 圆角体系
统一使用偏大的圆角，避免旧式硬边矩形：

- 小组件：`14px ~ 16px`
- 输入框 / 按钮：`18px`
- 卡片：`22px ~ 26px`
- 页面主容器 / 大面板：`30px ~ 32px`

### 4.2 边框规则
- 边框必须轻，不要生硬
- 推荐 1px 浅蓝白边框
- 不要重描边，不要黑灰粗边

### 4.3 阴影规则
阴影必须柔和、浅、低对比：

推荐：
- 卡片阴影：`0 10px 26px rgba(88,123,191,.08)`
- 浮层阴影：`0 18px 50px rgba(77,120,200,.12)`
- 主按钮阴影：`0 18px 38px rgba(49,103,243,.25)`

禁止：
- 又黑又硬的阴影
- 阴影扩散过大导致发脏
- 阴影方向混乱

---

## 5. 布局规范

### 5.1 总体布局
典型后台布局保持以下结构：

- 左侧导航栏
- 右侧主内容区
- 主内容区包含：
  - 顶部标题区
  - 用户信息 / 操作按钮
  - 数据概览卡片
  - 表格区
  - 待办区
  - 提醒区

### 5.2 留白原则
2026 风格的关键不是复杂，而是留白充足：

- 模块之间保持明显呼吸感
- 卡片内边距建议 `20px ~ 24px`
- 页面主区域横向间距保持统一
- 避免元素过于拥挤

### 5.3 对齐原则
- 所有标题、按钮、输入框、卡片边界要严格对齐
- 同类模块高度和节奏尽量统一
- 视觉重心要稳定，不可东倒西歪

---

## 6. 组件设计规范

### 6.1 卡片
卡片是整个系统的核心视觉单位。

要求：
- 白色或浅蓝白背景
- 细边框
- 大圆角
- 柔和阴影
- 可有极轻微渐变
- 可有微弱高光或 radial glow

禁止：
- 纯灰底传统后台卡片
- 过多花纹
- 强烈拟物

### 6.2 按钮
#### 主按钮
- 蓝色渐变背景
- 白色文字
- 高度统一
- 圆角偏大
- 有柔和蓝色阴影

建议：
- 背景：`linear-gradient(180deg, #5C93FF, #3167F3)`

#### 次按钮
- 白底
- 浅边框
- 深蓝灰文字
- 保留轻微阴影

### 6.3 输入框 / 搜索框
- 高度比传统后台更大
- 圆角 `18px`
- 白底 + 浅蓝边框
- 图标使用线性 SVG
- 可在 focus 态增加浅蓝外发光

### 6.4 表格
表格必须弱化“传统管理系统味道”，提升通透感。

要求：
- 表头浅蓝白底
- 行高舒展
- 分割线更轻
- hover 态轻微提亮
- 文本层级明确
- 操作列用品牌蓝强调

禁止：
- 过重的横线
- 深灰表头
- 密集型压迫排版

### 6.5 标签 / 优先级 Tag
- 使用胶囊形态
- 颜色柔和，不要纯色硬块
- 高优先：浅粉底 + 红字
- 低优先：浅青底 + 绿字

### 6.6 用户区
- 用户卡片保持圆角白底
- 头像可使用浅蓝圆形或圆角方形容器
- 信息展示简洁，避免复杂下拉造型

---

## 7. 图标规范

### 7.1 图标风格
统一使用 **简洁、线性、轻拟态感的 SVG 图标**。

要求：
- 线条简洁
- 转角圆润
- 粗细统一
- 业务语义清晰
- 风格统一

### 7.2 图标使用层级
图标分为三层：

#### A. 导航图标
- 小尺寸
- 线性
- 放在浅蓝白小底板里
- 和文字共同构成导航项

#### B. 功能图标
- 用于按钮、输入框、用户区、待办区
- 保持轻量、清晰

#### C. 装饰型业务图标
- 用于统计卡片右下角的大图标
- 使用更淡的蓝色插画感
- 不喧宾夺主
- 应接近业务含义，例如：
  - 客户总数：多人轮廓
  - 借贷单：文件 + 金额
  - 今日回款：饼图 / 时间 / 回款意象
  - 资料缺口：文件 / 信封 / 文档

### 7.3 禁止事项
- 禁止混用 emoji 图标
- 禁止一部分是线性，一部分是实心，一部分是 3D
- 禁止图标风格不统一
- 禁止用随机第三方 icon 导致气质不一致

---

## 8. 背景与装饰规范

### 8.1 背景
允许加入轻微科技感背景元素：

- 顶部柔和光轨
- 弧线
- 微弱星点
- 低透明度光晕
- 轻量数据轨迹感

原则：
- 装饰必须服务于高级感
- 不能影响可读性
- 不能压过业务内容

### 8.2 页面装饰插画
允许在以下区域加入装饰：

- 登录页左侧品牌区
- 侧边栏底部
- 统计卡片右下角

装饰风格要求：
- 统一浅蓝体系
- 低对比
- 科技感
- 半透明
- 不喧宾夺主

---

## 9. 页面级规范

### 9.1 登录页
结构建议：
- 左侧品牌展示区
- 右侧登录面板

要求：
- 左侧体现平台品牌感和科技感
- 右侧保持简洁、可信赖、易登录
- 登录框尺寸较大
- 标题要有足够气场
- 不要做成传统廉价后台登录页

### 9.2 工作台 / 仪表盘
要求：
- 顶部标题区要大气
- 数据卡片信息密度适中
- 表格和侧边待办形成稳定布局
- 右侧提醒区不宜过重
- 页面层次从上到下应清晰且有节奏

---

## 10. 代码实现规范（给 Agent / 前端）

### 10.1 技术要求
- 直接输出完整 HTML5
- CSS 可内联，也可拆分
- 优先原生 HTML + CSS + 少量 JS
- 非必要不引入复杂依赖
- 图标优先使用内联 SVG
- 装饰优先用 CSS / SVG 实现，不依赖位图

### 10.2 结构要求
- 语义化标签清晰
- class 命名统一
- 布局层、组件层、装饰层分离
- 可维护性优先

### 10.3 响应式要求
默认桌面端优先，但至少兼容基础响应式：
- 宽屏桌面为主
- 中等宽度下卡片可换行
- 小屏下允许模块堆叠

---

## 11. Agent 执行规则

当 Agent 基于本 design.md 输出页面时，必须遵守：

1. **先保业务结构，再提视觉品质**
2. **同一套页面必须保持同一设计系统**
3. **优先提升精致度，而不是盲目增加装饰**
4. **图标必须统一**
5. **按钮、输入框、卡片、表格的圆角和边框必须统一**
6. **任何高光、渐变、阴影都必须克制**
7. **避免出现老旧后台风、Bootstrap 风、低质原型风**
8. 输出的代码应达到：
   - 可以直接预览
   - 可以继续交给前端落地
   - 不只是概念稿

---

## 12. 质量判断标准

如果输出页面满足以下条件，说明设计方向正确：

- 第一眼看上去更像高端企业 SaaS，而不是传统后台
- 页面明亮、轻盈、专业
- 浅蓝体系统一，不乱
- 图标和卡片足够精致
- 信息密度合理，不拥挤
- 装饰存在但不抢内容
- 与同产品其他页面放在一起时没有割裂感

如果出现以下问题，说明需要返工：

- 看起来像旧式管理后台
- 图标杂乱
- 阴影太重
- 边框太灰太硬
- 元素过挤
- 装饰太廉价
- 蓝色太艳或太脏
- 页面虽然“花”，但不高级

---

## 13. 给 Agent 的简短执行提示

请基于本规范输出页面时，始终遵循下面这段原则：

> 用统一的浅蓝企业 SaaS 设计语言，重构页面为 2026 年审美下更精致、更高级、更可落地的后台界面。保留原有业务信息架构和主要文案，不随意删减模块。重点提升卡片层次、图标统一性、间距、阴影、边框、背景高光、按钮质感和表格精细度，输出结果必须是可以直接预览和继续开发的高保真 HTML 页面。
